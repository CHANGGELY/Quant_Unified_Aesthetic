# Product Guidelines

## Coding Standards (代码规范)
*   **中文化编程**：强制执行中文化命名。函数名（如 `def 计算账户余额()`）、变量名（如 `当前持仓量 = 1.5`）以及类名必须尽可能使用中文。
*   **防御性编程**：采用“卫语句 (Guard Clauses)”提前拦截异常。严禁“吞掉”错误，所有 API 调用和交易指令必须有完善的 Try-Catch 机制。
*   **中文注释头**：每个文件开头必须用通俗的中文解释该文件的核心职责。
*   **异步优先**：涉及网络 IO（行情获取、下单）的代码必须使用 `async/await` 异步架构。

## UI/UX Principles (设计准则)
*   **专业数据看板**：界面风格对标币安/彭博。信息密度优先，支持多窗口、大屏指标监控，确保核心交易数据（如深度图、仓位变化）一目了然。
*   **实时反馈**：任何操作必须在终端或界面上有实时的日志输出，让用户明确感知系统正在执行的任务。

## Communication Style (沟通风格)
*   **金牌编程导师**：严禁直接堆砌技术名词。遇到复杂概念（如 Webhook, WebSocket, 回测过拟合）时，必须用类比和通俗的语言进行解释。
*   **决策透明**：在执行关键的删除或修改操作前，必须说明原因及潜在风险。
