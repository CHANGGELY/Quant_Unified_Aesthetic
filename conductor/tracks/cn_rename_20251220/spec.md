# Track Specification: 核心目录与文件名全面中文化重构

## 1. Goal
将项目核心目录及文件重命名为中文，以符合“中文化编程”的产品准则，降低用户认知负荷，同时确保所有引用（Imports）、配置路径和脚本正常运行。

## 2. Scope
*   **重命名目录**: 
    *   `strategies/` -> `策略仓库/`
    *   `tests/` -> `测试用例/`
    *   `libs/` -> `基础库/`
    *   `apps/` -> `应用/`
    *   `services/` -> `服务/`
    *   `logs/` -> `系统日志/`
    *   `Quant_Unified/` 根目录下的其它关键英文文件。
*   **代码修正**: 
    *   全局搜索并替换所有涉及上述路径的 Python `import` 语句。
    *   更新 `sys.path` 路径配置（如 `sitecustomize.py`）。
    *   更新 `README.md` 和其他文档中的路径引用。
*   **验证**: 
    *   确保 Python 后端服务能正常启动。
    *   确保现有测试用例通过。

## 3. Non-Goals
*   暂时不修改第三方库（如 venv）生成的标准目录。
*   暂时不翻译代码内部的变量名（专注于文件系统层面的重构）。

## 4. Risks & Mitigation
*   **风险**: 循环引用或动态导入（`importlib`）失效。
*   **对策**: 使用 `grep` 全局搜索字符串引用，不仅仅是搜索 `import` 语句。
*   **风险**: Git 历史丢失。
*   **对策**: 使用 `git mv` 命令进行重命名，保留提交历史。
